Hybrid Profanity Detection & Review Rewriting System(Sentiment analysis and LLM finetuning)

QuestionPrep: AI-Based Interview Simulation Platform (FYP)

  • Built a three-stage answer evaluation pipeline: rapidfuzz bidirectional partial-ratio (lexical) → BGE-small-en bi-encoder (semantic similarity) → DeBERTa cross-encoder NLI (entailment/contradiction detection), with a length-ratio guard that penalises cut-off answers proportionally and a contradiction hard-cap that overrides high bi-encoder scores.
  • Deployed TCN-based stress detection entirely in the browser: an AudioWorklet processor captures raw PCM into a ring buffer, a Web Worker computes log-mel spectrograms in pure JS, and ONNX Runtime Web runs inference with WASM pre-cached by the service worker for offline use.
  • Built a RAG ingestion pipeline using Docling OCR → a mask-unmask MarkdownChunker (preserving code, math, and table blocks) → ChunkEvaluator → two-pass LFM2.5-1.2B enrichment (metadata + Q/A generation) → SimHash deduplication → BGE embeddings stored in ChromaDB, all orchestrated through FastAPI background tasks with SSE-based real-time progress streaming.
